## Key Decisions

- **No new How Might We statements** — mobile UX audit design exploration already exists for identified problems. Focus: scope each problem and link to existing work.
- **Half-sheet first, minimised state later** — half-sheet (Jay's live chat work) can be leveraged for help assistant, but minimised state (content-shifting) needs more design thinking before implementation. Prioritise half-sheet.
- **FAB rejected** — too prominent, covers content on scroll, inconsistent positioning. Not recommended.
- **Top-right placement** — recommended consistently across mobile (editor toolbar, home) and desktop. Emerging AI mental model (Slack pattern). Matthew: "it should always be there."
- **Hold icon redesign discussion for now** — valid idea (Japan robot icon = +7% engagement) but introducing it now risks derailing placement buy-in. One decision at a time.
- **HA + Canva AI component alignment underway** — Matthew working on shared input fields and visual consistency. Goal: same platform, different intents — accessed via smooth handoff, not embedded capabilities.

## Key Insights / Data

- **Japan icon change** — Matthew replaced question mark with robot icon; engagement increased ~7%. Strong signal for icon evolution.
- **User research (Tahlia presenting soon)** — Users overwhelmingly use ChatGPT (not Claude) to guide Canva usage. Many didn't know Canva had its own help assistant until shown. On discovery: "oh, you have your own ChatGPT."
- **User expectation gap** — Users expect image upload and design generation in HA, similar to ChatGPT/Claude workflows. Discovery gap: users don't know HA uses AI.
- **Strategic concept (Pri)** — Pre-editor design generation: user generates first draft via HA (in HC or editor) → hands off to Canva AI for continued editing. Targets users currently using ChatGPT/Claude for doc/presentation research → generate preview → open Canva.
